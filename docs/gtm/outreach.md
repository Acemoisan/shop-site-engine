# Outreach & Intake (value-first / free-audit play)

**Date:** 2026-06-15 · **Status:** draft templates for Phases 3–4.
**Verified principle:** value-first outreach that leads with a free analysis of the prospect's online presence **converts better than generic cold outreach** (3-0 verified). So every first touch leads with a specific, useful observation — never a generic pitch.

> The [AI site-audit tool](../research/2026-06-15-engagement-scoping-rubric.md) (PageSpeed API + SEOmator) is what makes this scale: it auto-generates a per-prospect 1-page audit that *is* the outreach hook.

---

## The prospecting funnel

```
1. SCRAPE   → Outscraper / Targetron: Calgary shops, filter "no website" or weak site
2. QUALIFY  → brick-and-mortar fit · sub-4★ rating (room to improve) · target vertical · has NAP
3. AUDIT    → run the audit tool on existing site (or note "no site / social-only")
4. REACH    → value-first message leading with one specific finding + the free 1-page audit
5. FOLLOW   → 1–2 light follow-ups referencing a second finding
6. INTAKE   → reply → short qualification → scope → quote (package or tune-up)
```

## Qualification checklist (before spending effort)

- [ ] **Brick-and-mortar local Calgary shop** (our fit) — not a chain, not online-only.
- [ ] **No website, or a weak one** (not mobile, slow, dated, broken) — there's a real problem to solve.
- [ ] **Target vertical** (salon/barber, restaurant/café, trades, retail) where we have a section kit.
- [ ] **Reachable** — has NAP / email / DM / phone.
- [ ] **Signal of being active** (recent reviews, posts) — they care about the business.
- [ ] *Bonus:* sub-4★ rating or visibly losing to competitors → sharper pain.

## The free 1-page audit (the hook) — structure

Auto-generated per prospect, then lightly humanized. One page, skimmable, specific.

1. **Header** — "[Shop Name] — Website & Online Presence Audit"
2. **Snapshot** — overall grade (A–F from the audit tool) + 1-line verdict.
3. **Top 3 issues** — each: the problem, why it costs them customers, the fix. (e.g. slow mobile load → lost calls; no menu schema → not in search; missing GBP fields → lower Maps rank.)
4. **What a fixed version looks like** — 2–3 bullets tied to their vertical (booking, click-to-call, reviews, menu).
5. **Soft CTA** — "Want me to build it? One-time project, you own everything. Reply and I'll send a quote."

> Honesty rule: if the audit grades **A** (already good), say so and move on. Costs nothing, builds reputation and referrals.

## Intake (when they reply)

Short, fast, scope-focused. 5–7 questions:

1. What does the shop do, and what's the #1 action you want website visitors to take? (call / book / order / visit)
2. Do you have a website now? (URL) — or starting fresh?
3. Which do you need: brochure, bookings, online ordering, or e-commerce?
4. Do you have a logo, photos, and copy — or need help creating them?
5. Do you own your domain already? (and email on it?)
6. Timeline / any launch date?
7. Rough budget sense (or present the tiers and let them pick).

→ Map to a [package or tune-up](packaging.md), confirm scope boundary (content self-edit vs structural = paid), send quote + deposit terms.

## Expectation-setting (say up front, avoids friction later)

- **One-time fee, no monthly fees to us.** You own the domain, hosting, and editor accounts.
- **You can edit your own content** (text, photos, hours, menu) anytime via a simple visual editor — we train you.
- **Design/layout changes** after launch are a separate small job (keeps your price low).
- Bookings/payments run on **your** Square/Stripe/Shopify account (your fees, your money).
- Deposit to start (e.g. 50%), balance on launch. *(Owner to set terms.)*
